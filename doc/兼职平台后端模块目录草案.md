# 兼职平台后端模块目录草案

> 目标：保留若依框架层不动，仅重构兼职平台业务层目录。

## 1) 框架不动区

以下目录保持不动（仅使用，不改结构）：

- `backend/ruoyi-common`
- `backend/ruoyi-framework`
- `backend/ruoyi-system` 中 `Sys*` 平台能力
- `backend/ruoyi-admin` 中系统基础控制器（登录、监控、工具等）

## 4) 目录创建顺序（建议）

1. 先建 `ruoyi-system` 业务包骨架（`common` -> `recruitment` -> `student` -> `enterprise` -> `admin/audit`）。
2. 再建 `mapper/biz` 对应 XML 目录。
3. 再建 `ruoyi-admin` 新控制器目录（`student`、`enterprise`、`admin/audit`）。
4. `backend/sql` 版本化目录最后统一创建并迁入历史脚本。

## 5) 命名规范（建议）

- Java 包名：全小写、无连字符，如 `com.ruoyi.biz.admin.audit`。
- 类名：角色前缀 + 业务对象 + 后缀，如 `EnterprisePostController`。
- DTO/VO：`*Request`、`*Response` 或 `*DTO`、`*VO` 二选一并保持一致。
- 权限前缀：`admin:audit:*`、`enterprise:*`、`student:*`。
- SQL 文件名：`VYYYYMMDD_NN__desc.sql`，回滚文件 `RYYYYMMDD_NN__desc.sql`。

## 6) 路由收敛约定（不保留旧兼容）

- 旧路径（如 `/campus/*`、旧 `/system/*` 业务接口）不再保留兼容委派。
- 业务接口统一使用 `student/*`、`enterprise/*`、`admin/audit/*` 新路径。
- 新增业务仅写入新模块目录，不再向旧目录加代码。
