# 企业端前后端改造总结

> 改造日期：2026-05-06
> 改造范围：企业端全部 4 个业务模块（资料、岗位、投递、评价）

## 一、后端 Bug 修复

### 1.1 N+1 查询修复（HIGH）

**文件**：`EnterpriseFlowServiceImpl.java`、`JobApplicationMapper.java`、`JobApplicationMapper.xml`

**问题**：`listMyApplications()` 先查全表，再对每条记录调用 `isApplicationOwnedByEnterprise()` 发起单独查询。10000 条投递记录会产生 10001 条 SQL。

**修复**：在 `JobApplicationMapper.xml` 新增 `selectApplicationListByEnterpriseId`，通过 `INNER JOIN t_job_post` 在数据库层按 `enterprise_id` 过滤，一条 SQL 完成。

```sql
select a.* from t_job_application a
inner join t_job_post p on a.post_id = p.post_id and p.enterprise_id = #{enterpriseId} and p.del_flag = '0'
```

### 1.2 提交审核无状态检查（MEDIUM）

**文件**：`EnterpriseFlowServiceImpl.java` → `submitMyProfileAudit()`

**问题**：已通过审核的企业可重复提交，意外将状态重置为"待审核"。

**修复**：增加前置状态检查：
- `auditStatus == "1"`（已通过）→ 抛出"审核已通过，无需重复提交"
- `auditStatus == "0"`（待审核）→ 抛出"审核正在处理中，请勿重复提交"
- 仅 `null` / `"2"`（驳回）时允许重新提交

### 1.3 Controller 注解缺失（MEDIUM）

**文件**：全部 4 个企业端 Controller

**问题**：缺少 `@Validated`（无 JSR-303 校验）和 `@Log`（无操作审计日志）。

**修复**：参照 `SysNoticeController` 标准补齐：
- 所有 `@RequestBody` 参数加 `@Validated`
- 所有写操作（POST/PUT/DELETE）加 `@Log(title = "...", businessType = BusinessType.XXX)`

### 1.4 评分校验缺失（MEDIUM）

**文件**：`EnterpriseEvaluationController.java`

**问题**：无评分范围校验，可提交 0、-999 等非法值。

**修复**：`EnterpriseEvaluationRequest` 内部类加 `@NotNull`、`@Min(1)`、`@Max(5)` 注解（使用 `jakarta.validation.constraints`，Spring Boot 4.x 命名空间）。

### 1.5 乱码注释（LOW）

**文件**：`EnterpriseInfo.java` 第 35 行

**修复**：`/** 鑱旂郴鐢佃瘽 */` → `/** 联系电话 */`

## 二、后端新增功能

### 2.1 岗位删除接口

| 项目 | 内容 |
|---|---|
| 端点 | `DELETE /enterprise/post/{postId}` |
| 权限 | `enterprise:post:remove` |
| 实现 | 软删除，设置 `del_flag = '2'` |
| 涉及文件 | `IEnterpriseFlowService.java`、`EnterpriseFlowServiceImpl.java`、`EnterprisePostController.java` |

### 2.2 投递拒绝接口

| 项目 | 内容 |
|---|---|
| 端点 | `PUT /enterprise/application/{applicationId}/reject` |
| 权限 | `enterprise:application:reject` |
| 实现 | 状态从 `"0"`（已投递）变为 `"3"`（已拒绝），记录 `enterpriseRemark` |
| 涉及文件 | `IEnterpriseFlowService.java`、`EnterpriseFlowServiceImpl.java`、`EnterpriseApplicationController.java`、`IJobApplicationService.java`、`JobApplicationServiceImpl.java` |

**注意**：`enterprise:application:reject` 权限点需在数据库 `sys_menu` 表中手动添加，并关联到企业角色。
