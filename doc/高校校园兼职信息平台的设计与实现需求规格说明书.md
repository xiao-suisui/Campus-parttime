# 一、引言

## 1.1 项目背景

当前高校校园兼职市场存在三大核心痛点：一是信息不对称，学生获取兼职渠道分散，虚假岗位、黑中介乱象频发，学生权益难以保障；二是供需对接效率低，企业发布校园兼职缺乏精准触达渠道，学生求职缺乏高效的筛选匹配工具，双方沟通成本高；三是兼职记录零散难追溯，学生兼职经历无统一归档渠道，企业无法快速核验学生过往兼职履历，缺乏有效的信用约束机制。

为解决上述问题，拟开发一套面向高校学生、用工企业的智能化校园兼职信息平台，打通学生求职、企业招聘、平台监管的全流程，为学生提供安全、便捷、精准的兼职信息服务，为企业提供高效的校园人才对接渠道。

## 1.2 项目意义

1. 实践意义：基于成熟开源框架实现全栈开发，覆盖需求分析、系统设计、编码实现、测试部署的完整软件开发流程，符合本科毕业设计的能力培养目标。

2. 应用价值：解决校园兼职市场的核心痛点，通过资质审核、双向互评机制保障岗位真实性，通过智能推荐提升供需匹配效率，为高校学生提供安全可靠的兼职服务平台。

3. 技术价值：采用主流前后端分离架构，结合跨端开发技术，兼顾系统的稳定性、可扩展性与多端适配能力，为后续功能迭代预留充足空间。

## 1.3 文档范围

本文档为《高校校园兼职信息平台的设计与实现》毕业设计的核心需求文档，明确了系统的业务边界、功能需求、非功能需求、技术选型、数据库设计思路与开发计划，是系统设计、开发、测试与论文撰写的核心依据。

本文档不包含复杂三方API对接、商业化付费模块的需求，预留相关扩展接口，后续可根据开发进度迭代补充。

# 二、需求分析

## 2.1 可行性分析

### 2.1.1 技术可行性

本项目采用Spring Boot+Vue.js+MySQL的主流技术栈，基于成熟的RuoYi前后端分离框架二次开发，复用框架原生的权限管理、菜单管理、定时任务、数据校验等核心能力，无需从零搭建基础架构；学生端采用Uniapp跨端框架，一套代码实现多端适配，开发效率高，相关技术文档、开源资源丰富，完全满足毕业设计的开发需求与技术难度要求。

### 2.1.2 经济可行性

本项目所有核心框架均为开源免费技术，无商业版权费用；开发与部署可采用高校服务器、云厂商学生免费/低价服务器，硬件成本极低；无需对接付费三方API，无额外运营成本，完全符合毕业设计的经济预算要求。

### 2.1.3 操作可行性

平台学生端采用移动端原生交互逻辑，界面简洁直观，符合学生用户的操作习惯；企业端与管理员端基于RuoYi框架成熟的UI设计，操作流程标准化，学习成本低；核心业务流程闭环清晰，无需复杂培训即可快速上手。

### 2.1.4 时间可行性

本项目分阶段开发，核心业务闭环开发周期可控，预留充足的测试、优化与论文撰写时间，完全匹配本科毕业设计的时间周期要求。

## 2.2 用户角色与核心诉求

本平台核心分为三类用户角色，各角色的定位与核心诉求如下：

|角色|定位|核心诉求|权限边界|
|---|---|---|---|
|学生用户|平台服务核心对象，兼职求职者|快速找到安全靠谱的兼职岗位、便捷管理简历与投递记录、归档兼职履历|注册无需审核，可使用平台所有求职相关功能|
|企业用户|兼职岗位提供方，用工主体|高效发布兼职岗位、精准筛选优质学生、管理招聘全流程|注册后需管理员审核资质，审核通过后方可发布岗位、管理简历|
|系统管理员|平台监管方与运营方|审核企业资质、监管平台内容、保障平台合规运行、统计运营数据|拥有平台最高管理权限，复用RuoYi框架原生系统管理能力|
## 2.3 核心业务流程

### 2.3.1 企业入驻与资质审核流程

企业注册账号→填写企业资质信息+上传营业执照→提交审核申请→管理员后台审核→审核通过→开放岗位发布等核心权限；审核驳回→企业修改资质信息→重新提交审核。

### 2.3.2 学生求职全流程

学生注册账号→完善个人简历→浏览/筛选/搜索岗位→岗位详情页收藏/投递简历→查看投递状态→企业录用→线下完成兼职→双方完成双向互评→兼职记录归档。

### 2.3.3 岗位发布与管理流程

资质审核通过的企业→发布岗位填写完整信息→提交后直接上架展示（预留岗位二次审核开关）→管理岗位上下架/编辑/延期→查看岗位投递简历→处理简历申请→录用学生→完成兼职后互评。

### 2.3.4 平台管理流程

管理员登录系统→企业资质审核→用户账号管理→违规岗位监管→平台公告发布→运营数据统计→系统配置管理。

# 三、系统总体设计

## 3.1 总体架构设计

本系统采用经典的前后端分离架构，整体分为四层，架构设计如下：

1. 接入层：学生端通过Uniapp编译后的H5/微信小程序接入，企业端与管理员端通过PC端Web后台接入，统一通过HTTP/HTTPS协议请求后端接口。

2. 应用层：基于Spring Boot开发后端服务，分为用户管理模块、资质审核模块、岗位管理模块、简历投递模块、互评模块、智能推荐模块、运营统计模块、系统管理模块，完全复用RuoYi框架原生的权限控制、定时任务、日志管理等基础能力。

3. 数据层：采用MySQL作为核心关系型数据库，存储所有业务数据与系统配置数据；预留Redis缓存接入能力，后续可根据开发进度补充。

4. 基础设施层：支持部署在高校私有服务器或公有云服务器，提供数据备份、日志存储、监控告警等基础能力。

## 3.2 技术选型

|系统层级|技术选型|选型说明|
|---|---|---|
|后端开发|Spring Boot 2.7+/3.x + RuoYi 最新前后端分离版|基于成熟开源框架二次开发，复用原生权限、定时任务等能力，降低开发成本|
|前端开发（PC后台）|Vue.js + Element UI|与RuoYi框架原生技术栈一致，无需额外改造，适配企业端与管理员端需求|
|前端开发（学生移动端）|Uniapp + uView UI|一套代码实现多端适配，支持H5、微信小程序等主流端，符合毕业设计演示需求|
|数据库|MySQL 8.0|主流关系型数据库，开源免费，适配RuoYi框架，满足业务数据存储需求|
|开发工具|IDEA、HBuilderX、Navicat、Postman|主流开发调试工具，适配全栈开发流程|
## 3.3 功能模块总体划分

本系统整体分为三大端，共10个核心功能模块，模块划分如下：

1. 学生端（Uniapp跨端）：账号管理模块、简历管理模块、岗位大厅模块、智能推荐模块、投递与收藏管理模块、兼职与互评模块、公告查看模块。

2. 企业端（RuoYi PC后台）：账号与资质管理模块、岗位管理模块、简历管理模块、兼职与互评模块、基础数据统计模块。

3. 管理员端（RuoYi PC后台）：资质审核模块、用户管理模块、岗位监管模块、公告管理模块、运营数据统计模块、系统管理模块（完全复用RuoYi原生能力）。

## 3.4 系统部署架构

本系统采用轻量化部署架构，适配毕业设计的部署需求：

- 后端服务、MySQL数据库统一部署在单台Linux服务器（云服务器学生机/高校服务器）；

- 学生端Uniapp编译后的H5包、PC后台前端包部署在服务器Nginx服务下，实现静态资源访问与接口反向代理；

- 预留集群部署扩展能力，后续可根据需求拆分部署。

# 四、核心功能模块详细需求

## 4.1 学生端（Uniapp跨端）功能需求

学生端为Uniapp开发的跨端应用，核心实现求职全流程功能，一套代码兼容H5、微信小程序等多端，核心功能如下：

### 4.1.1 账号管理模块

- 注册功能：支持手机号+密码注册，必填字段包括姓名、学号、学校、专业，注册成功后无需审核，可直接登录使用；

- 登录功能：支持手机号+密码登录，预留短信验证码登录接口；

- 账号管理：支持个人信息修改、登录密码修改、忘记密码重置、账号退出登录。

### 4.1.2 简历管理模块

- 提供标准化简历模板，支持学生在线编辑简历，必填字段包括：基本信息、教育经历、兼职经历、技能特长、求职意向、期望薪资；

- 支持简历预览功能，投递岗位时自动关联最新版简历；

- 预留附件简历上传接口，后续可根据开发进度补充。

### 4.1.3 岗位大厅模块

- 岗位列表展示：展示所有上架中的岗位，默认按发布时间倒序排序，支持按薪资从高到低、热度排序；

- 多条件筛选：支持按薪资范围、结算方式（日结/周结/月结）、工作地点（校内/校外/远程）、工作类型（短期/长期/实习）、发布时间进行筛选；

- 关键词搜索：支持按岗位名称、企业名称、岗位要求关键词搜索岗位；

- 岗位详情页：展示岗位完整信息，包括岗位名称、薪资范围、结算方式、工作地点、工作时间、岗位要求、招聘人数、截止时间、企业信息、联系方式，明确标注平台不介入薪资结算，双方线下沟通对接。

### 4.1.4 智能推荐模块（毕设创新点1）

- 首页设置专属推荐专区，基于规则实现个性化岗位推荐；

- 核心匹配逻辑：为岗位打上类型、行业、专业要求、地点等标签，为学生打上专业、求职意向、历史投递/收藏岗位的标签，通过标签匹配度排序，为学生推荐高匹配度的岗位；

- 预留协同过滤算法扩展接口，后续可根据开发进度优化推荐逻辑。

### 4.1.5 投递与收藏管理模块

- 岗位收藏：支持在岗位详情页收藏/取消收藏岗位，收藏列表支持快速投递、取消收藏；

- 简历投递：支持在岗位详情页一键投递简历，投递后不可重复投递同一岗位；

- 投递记录管理：展示所有投递记录，支持按投递状态筛选，状态包括：待查看、已查看、初筛通过、已拒绝、已录用，实时同步企业端的处理状态。

### 4.1.6 兼职与互评模块（毕设创新点2）

- 我的兼职：展示状态为“已录用”的兼职记录，企业标记兼职完成后，可进入互评环节；

- 双向互评：兼职完成后，学生可对对应企业进行1-5分评分与文字评价，同时可查看企业对自己的评价；

- 评价展示：学生的评价会展示在企业详情页，企业的评价会展示在学生简历中，形成双向信用约束。

### 4.1.7 其他辅助模块

- 平台公告：查看管理员发布的平台公告、兼职安全提醒；

- 岗位举报：【预留扩展】支持对虚假/违规岗位进行举报，提交管理员处理；

- 黑名单：【预留扩展】支持屏蔽违规企业的岗位展示。

## 4.2 企业端（RuoYi PC后台）功能需求

企业端基于RuoYi框架二次开发，与管理员端共用同一后台，通过RBAC权限控制实现菜单与接口隔离，核心功能如下：

### 4.2.1 账号与资质管理模块

- 企业注册：支持企业账号注册，必填字段包括企业名称、统一社会信用代码、法人信息、联系人、联系电话，需上传营业执照图片；

- 资质审核管理：注册成功后可登录后台，仅可查看资质审核状态，无其他业务权限；审核驳回后，可修改资质信息重新提交审核；审核通过后，开放所有业务权限；

- 账号管理：支持企业基本信息修改、登录密码修改、企业简介编辑（展示在学生端企业详情页）。

### 4.2.2 岗位管理模块

- 岗位发布：支持发布兼职岗位，必填字段包括岗位名称、薪资范围、结算方式、工作地点、工作类型、工作时间、岗位要求、招聘人数、截止时间、联系方式；

- 岗位列表管理：支持查看所有已发布的岗位，可进行编辑、上架、下架、删除、延期操作，展示岗位的浏览量、收藏量、投递量数据；

- 【预留扩展】支持岗位置顶、岗位刷新功能。

### 4.2.3 简历管理模块

- 简历列表：展示所有收到的学生简历，支持按所属岗位、投递时间、处理状态筛选；

- 简历处理：支持查看学生简历详情，对简历进行状态标记，包括：已查看、初筛通过、已拒绝、已录用，状态变更实时同步至学生端；

- 【预留扩展】支持简历批量处理、简历导出功能。

### 4.2.4 兼职与互评模块

- 兼职管理：展示所有已录用的学生记录，支持标记兼职“已完成”，触发互评环节；

- 学生评价：兼职完成后，可对学生进行1-5分评分与文字评价，查看学生对企业的评价；

- 【预留扩展】支持学生履约记录归档功能。

### 4.2.5 基础数据统计模块

- 核心数据看板：统计企业的岗位发布总数、累计投递量、录用人数、完成兼职数，支持按时间范围筛选；

- 【预留扩展】支持岗位数据趋势图表、数据导出功能。

## 4.3 管理员端（RuoYi PC后台）功能需求

管理员端完全复用RuoYi框架原生的系统管理能力，仅新增业务管理模块，无需对框架核心进行改造，核心功能如下：

### 4.3.1 企业资质审核模块

- 审核列表：展示所有企业提交的资质审核申请，支持按审核状态、企业名称、提交时间筛选；

- 审核操作：支持查看企业完整资质信息与营业执照图片，进行审核通过/驳回操作，驳回需填写明确的驳回原因；

- 审核记录：所有审核操作自动归档，支持历史记录查询。

### 4.3.2 用户管理模块

- 学生用户管理：展示全平台学生用户列表，支持按学校、姓名、学号、注册时间筛选，可查看用户详情、禁用/启用账号、重置密码；

- 企业用户管理：展示全平台企业用户列表，支持按审核状态、企业名称、注册时间筛选，可查看企业详情、禁用/启用账号、重置密码。

### 4.3.3 岗位监管模块

- 全平台岗位列表：展示所有企业发布的岗位，支持按企业、岗位状态、发布时间筛选；

- 违规处理：支持对虚假/违规岗位进行强制下架操作，查看岗位的举报记录【预留扩展】；

- 【预留扩展】支持岗位批量操作、岗位审核开关配置。

### 4.3.4 公告管理模块

- 公告发布：支持发布平台公告，必填字段包括公告标题、公告内容、公告类型、发布时间，支持设置是否置顶、下架时间；

- 公告管理：支持对已发布的公告进行编辑、删除、上下架操作，公告内容实时同步至学生端。

### 4.3.5 运营数据统计模块

- 全平台数据看板：展示核心运营指标，包括学生总数、企业总数、审核通过企业数、岗位发布总数、累计投递数、完成兼职总数；

- 数据可视化：基于ECharts实现近7天/30天的用户注册趋势、岗位发布趋势图表；

- 数据导出：支持核心运营数据导出为Excel文件。

### 4.3.6 系统管理模块
