# 移动端开发计划

> 生成日期：2026-05-06
> 前置依赖：PC 端全部功能已完成（阶段一~三）
> 技术栈：UniApp Vue 2 + uni-ui

## 本阶段目标

完成移动端（H5/小程序/App）的学生端全部业务页面，包括：
- 任务 4.1：移动端基础设施
- 任务 4.2：学生岗位大厅页
- 任务 4.3：学生投递管理页
- 任务 4.4：学生收藏管理页
- 任务 4.5：学生互评页
- 任务 4.6：学生个人中心

预计工作量：1-2 周

---

## 任务 4.1：移动端基础设施

**目标**：确保移动端能正常请求后端 API

**操作**：
- 检查 `RuoYi-App-master/config.js` 中 API 地址配置
- 检查请求封装（JWT token 注入、错误码处理）
- 确认登录/注册流程可对接后端 `/login` `/register`
- 确认 `uni_modules/` 中 uni-ui 组件可用

**涉及文件**：
- `RuoYi-App-master/config.js`
- `RuoYi-App-master/api/login.js`
- `RuoYi-App-master/utils/request.js`（如有）

---

## 任务 4.2：学生岗位大厅页

**目标**：岗位列表 + 筛选 + 搜索 + 详情

**页面设计**：
- 顶部搜索栏（关键词搜索）
- 筛选条件（薪资范围、结算方式、工作地点类型、工作类型）
- 岗位卡片列表（下拉刷新 + 上拉加载更多）
- 岗位详情页（完整信息 + 收藏按钮 + 投递按钮）

**API 对接**：
- `GET /student/post/list` — 岗位列表
- `GET /student/post/{postId}` — 岗位详情
- `POST /student/collection` — 收藏
- `POST /student/application` — 投递

**涉及文件**：
- 新建页面：`pages/student/post/list.vue`、`pages/student/post/detail.vue`
- 修改路由：`pages.json`

## 实施顺序

```
1. 4.1 基础设施 — 确保请求链路通畅
2. 4.2 岗位大厅 — 核心页面，优先级最高
3. 4.3 投递管理 — 与岗位大厅关联
4. 4.4 收藏管理 — 简单列表页
5. 4.5 互评页 — 独立功能
6. 4.6 个人中心 — 收尾
```

## 注意事项

- 使用 `uni.request` 封装，参考现有 `api/login.js` 的写法
- 页面路由在 `pages.json` 中注册
- Token 存储使用 `uni.setStorageSync` / `uni.getStorageSync`
- 下拉刷新使用 `onPullDownRefresh`，上拉加载使用 `onReachBottom`
- UI 组件使用 `uni_modules/uni-ui` 中已安装的组件
