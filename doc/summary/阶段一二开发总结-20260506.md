## 四、关键设计决策

1. **简历/信息管理走 FlowService 两层架构**：Controller 不直接调用 CRUD Service，与现有架构保持一致
2. **Upsert 模式**：save 方法统一处理新增和修改，通过查询现有记录判断是 insert 还是 update，保留 status/delFlag 等系统字段
3. **岗位监管的企业名称关联**：在 Controller 层批量查询企业名称，通过 JobReviewVO 内部类扩展 JobPost，避免修改 Domain 层
4. **运营统计的趋势数据**：使用反射读取 createTime 字段按日期分组，避免修改 Mapper XML，适合毕业项目数据量
