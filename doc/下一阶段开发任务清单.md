## 阶段一：学生端补齐（预计 2-3 天）

### 任务 1.1：学生简历 Controller + 前端页面

**目标**：实现简历 CRUD，替换当前占位页

**后端**（数据层已就绪，仅需加 Controller）：
- 新建 `backend/ruoyi-admin/.../controller/student/StudentResumeController.java`
- 端点设计（参照企业端 Controller 风格）：
  - `GET /student/resume` — 查询当前学生简历
  - `POST /student/resume` — 新增简历
  - `PUT /student/resume` — 修改简历
- 注入 `IStudentFlowService` 或直接注入 `IStudentResumeService`
- 加 `@Validated`、`@Log`、`@PreAuthorize("student:resume:*")`

**前端**：
- 新建 `frontend/src/api/student/resume.js` — API 文件
- 重写 `frontend/src/views/system/student/resume/index.vue` — 简历编辑表单
- 表单字段：求职意向、期望薪资、教育经历、兼职经历、技能特长、自我介绍
- 参照 `enterprise/profile/index.vue` 的 Card + Form 布局

**SQL**：
- 新增 RBAC 权限：`student:resume:query`、`student:resume:add`、`student:resume:edit`
- 新增菜单项绑定到学生角色

**涉及已有文件**：
- `backend/ruoyi-system/.../student/service/IStudentResumeService.java`（已存在）
- `backend/ruoyi-system/.../student/domain/StudentResume.java`（已存在）
- `backend/ruoyi-system/.../resources/mapper/biz/student/StudentResumeMapper.xml`（已存在）

## 阶段二：管理员端补齐（预计 2-3 天）

### 任务 2.1：岗位监管模块

**目标**：管理员可查看全平台岗位并强制下架违规岗位

**后端**：
- 新建 `backend/ruoyi-admin/.../controller/admin/AdminJobReviewController.java`
- 端点设计：
  - `GET /admin/job-review/list` — 分页查询全平台岗位（关联企业名称）
  - `GET /admin/job-review/{postId}` — 岗位详情
  - `PUT /admin/job-review/{postId}/force-down` — 强制下架
- 注入 `IJobPostService`，查询时不按 `enterpriseId` 过滤
- 强制下架直接设置 `postStatus = "2"`

**前端**：
- 新建 `frontend/src/api/admin/jobReview.js` — API 文件
- 新建 `frontend/src/views/system/admin/jobReview/index.vue` — 岗位列表 + 强制下架
- 表格列：岗位名称、企业名称、薪资、状态、投递量、发布时间
- 操作按钮：详情、强制下架

**SQL**：
- 整合旧脚本中 `campus:job:review:*` 权限到新 RBAC 脚本
- 权限点：`admin:job-review:list`、`admin:job-review:query`、`admin:job-review:force-down`

**关键实现细节**：
- 需要 `JobPostMapper.xml` 新增一个 JOIN 查询，关联 `t_enterprise_info` 获取企业名称
- 或者在前端通过企业 ID 单独查询企业名称

## 阶段三：智能推荐模块（预计 2-3 天，创新点 1）

### 任务 3.1：岗位标签体系

**目标**：为岗位和学生建立标签，实现基于规则的匹配推荐

**数据库**：
- 新建 `t_job_tag` 表（tag_id, tag_name, tag_type, create_time）
- 新建 `t_post_tag_rel` 表（post_id, tag_id）
- 新建 `t_student_tag_rel` 表（student_id, tag_id）
- 或者简化方案：在 `t_job_post` 和 `t_student_info` 中加 JSON 字段存储标签

**后端**：
- 新建 `backend/ruoyi-system/.../biz/recommend/` 包
- 新建 `RecommendService.java` + `RecommendServiceImpl.java`
- 核心方法：`List<JobPost> recommendPosts(Long studentId, int limit)`
- 匹配逻辑：
  1. 读取学生标签（专业、求职意向、历史投递/收藏的岗位类型）
  2. 查询上架岗位，按标签匹配度打分排序
  3. 匹配维度：专业相关性、地点偏好、薪资范围、工作类型
- 新建 `StudentJobController` 中增加 `GET /student/post/recommend` 端点

**前端**：
- 在学生岗位浏览页 `post/index.vue` 增加"推荐"标签页或推荐区域
- 新建 `frontend/src/api/student/recommend.js`
